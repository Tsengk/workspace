# 玫瑰病害分类模型 — 创新方案

## 当前基线

| 项 | 现状 |
|---|------|
| 方法 | 文件夹名 = 类别标签，YOLOv8n-cls 直接训练 |
| 数据 | 40,679 张（8 类，train 32,543 / val 8,136） |
| 准确率 | 预估 80-85%（Top-1） |
| 问题 | 类别不平衡（fresh_leaf 16K vs rose_mosaic 1K）、增强图与原始图混杂、无细粒度定位 |

## 方案一：数据增强策略优化

### 1.1 图像增强来源追溯拆分

**问题**：当前 `augmented_*` 和原始图混在同一数据集，同一原图的不同增强版可能分别落入 train 和 val，导致验证集准确率虚高（数据泄露）。

**方案**：按原图 MD5 哈希或文件名规律分组，确保同一原图的所有增强版本只出现在 train 或 val 其中一侧。

```
当前: leaf_001.jpg → train
      augmented_leaf_001.jpg → train  ✅
      augmented_v2_leaf_001.jpg → val   ❌ 泄露

修正: 以原图为单位拆分，所有变体归入同一集合
```

### 1.2 类别平衡采样

**问题**：fresh_leaf 是 rose_mosaic 的 16 倍，模型偏向多数类。

**方案**：训练时使用加权随机采样，每个 batch 中等概率抽取各类别。

```python
# YOLOv8 不支持内置平衡采样，可通过以下方式绕过：
# 1. 对少样本类别过采样（复制图片 + 强增强）
# 2. 对多样本类别欠采样（每 epoch 随机选 N 张）
# 3. 自定义 DataLoader 加权采样
```

**预期提升**：小类准确率 +5~10%

## 方案二：两阶段级联模型

### 思路

先用分类判断「有病/无病」，再对有病样本细分类。

```
输入图片
    │
    ▼
阶段1: 二分类 (fresh_leaf vs 有病)
    │
    ├── fresh_leaf → 输出: 健康
    │
    └── 有病 →
              │
              ▼
         阶段2: 7分类 (黑斑/霜霉/白粉/虫洞/虫害/花叶/锈病)
              │
              ▼
         输出: 具体病害
```

### 优势

| 指标 | 单阶段 8 分类 | 两阶段级联 |
|------|-------------|-----------|
| 健康叶误判率 | 较高（被误判为某病） | 低（先过滤） |
| 病害间混淆 | 7 类互相干扰 | 排除健康叶干扰后更准 |
| 部署灵活性 | 固定 | 阶段1可独立用于初筛 |

## 方案三：多尺度特征融合

### 思路

YOLOv8n-cls 默认只输出单一尺度分类结果。对于叶片病害，局部纹理（孢子、菌斑）和全局形态（叶片卷曲、变色范围）同样重要。

```
输入图片 640×640
    │
    ├── 全局分支: 下采样 → 全局特征（叶片形态、颜色分布）
    │
    ├── 局部分支: ROI 裁剪 → 局部特征（病斑纹理、孢子）
    │
    └── 融合 → 分类头
```

### 实现方式

- **方案A**：修改 YOLOv8 分类头，增加多尺度特征拼接
- **方案B**：Timm 库替换 backbone 为 EfficientNet/ConvNeXt，利用其内置多尺度结构
- **方案C（简单）**：不改模型，输入图片做多尺度裁剪后投票

```python
# 方案C 示例（不改模型，纯推理增强）
def predict_multiscale(image_path):
    crops = [
        image,                          # 原图
        image.resize(224, 224),         # 缩略图（全局）
        center_crop(image, 320),        # 中心裁剪（局部）
    ]
    votes = [model(c).probs.top1 for c in crops]
    return majority_vote(votes)
```

## 方案四：自监督预训练

### 思路

当前用 `yolov8n-cls.pt`（ImageNet 预训练），但 ImageNet 是通用物体分类，与叶片病害的纹理、颜色分布差异大。

**方案**：在 40K 未标注病害图片上做自监督预训练（MoCo / SimCLR / MAE），让 backbone 先学会叶片纹理特征，再微调分类任务。

```
39.7K 无标签数据 → 自监督预训练(MoCo v3) → 微调 8 分类
                                                  ↓
                                          预期提升 2-4%
```

| 步骤 | 说明 |
|------|------|
| 1. 自监督预训练 | 用 MoCo v3 或 DINO 在全部 40K 图上训练 200 epoch |
| 2. 替换 backbone | 将预训练权重加载到 YOLOv8 或 ResNet |
| 3. 微调分类 | 正常训练 50 epoch |

## 方案五：知识蒸馏（轻量化部署）

### 思路

用大模型（yolov8m-cls）作为 Teacher，训练小模型（yolov8n-cls）作为 Student，让小模型逼近大模型精度。

```
yolov8m-cls (Teacher, 高精度) ──软标签──→ yolov8n-cls (Student, 轻量)
                                              ↓
                                         部署到边缘设备
```

| 模型 | 参数量 | 推理速度(CPU) | 预期准确率 |
|------|-------|-------------|-----------|
| yolov8m-cls (Teacher) | 15M | 慢 | 88-90% |
| yolov8n-cls (Student 蒸馏) | 2.7M | 快 3x | 86-88% |
| yolov8n-cls (直接训练) | 2.7M | 快 3x | 80-85% |

## 方案六：细粒度病害分级

### 思路

当前白粉病只有单类别。可扩展为分级检测：

```
powdery_mildew/ 目录下实际可能包含不同严重程度 → 
按病斑面积占比自动分 3 级：轻度(<10%) / 中度(10-30%) / 重度(>30%)

实现：用大模型（如 SAM）自动分割病斑区域 → 计算面积比 → 自动分级标签
```
