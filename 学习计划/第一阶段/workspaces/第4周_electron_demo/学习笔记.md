## 笔记

**2026-06-30**

Electron 应用由两个进程组成：主进程（Main Process）和渲染进程（Renderer Process）。理解这两个进程的职责和通信方式，是 Electron 开发的核心。

### Electron 核心概念

1. **主进程**（main.js）
   - 唯一的，负责创建窗口、管理应用生命周期
   - 可直接访问 Node.js API（fs、path、dialog 等）
   - 通过 `ipcMain.handle()` 注册 IPC 处理器

2. **渲染进程**（renderer.js）
   - 每个窗口一个，负责界面展示和用户交互
   - 不能直接访问 Node.js API（安全隔离）
   - 通过 `window.electronAPI`（由 preload 暴露）调用主进程功能

3. **预加载脚本**（preload.js）
   - 在渲染进程加载前执行
   - 使用 `contextBridge.exposeInMainWorld()` 暴露安全 API

4. **IPC 通信**（进程间通信）
   - 渲染 → 主：`ipcRenderer.invoke('channel', ...args)`
   - 主 → 渲染：`ipcMain.handle('channel', handler)`

### 本 Demo 实现要点

- 主进程负责文件对话框（dialog）和文件读取（fs）
- 渲染进程负责日志解析、表格渲染、搜索过滤
- preload.js 只暴露两个方法：`file.open()` 和 `file.read()`
- 纯 HTML/CSS/JS，不依赖 React/Vue 构建工具
- 界面完全对齐 ICAOS_TOOL 暗色主题设计系统

### 日志解析策略

使用正则匹配 4 种常见日志格式：
- `[2024-01-15 10:30:45] [ERROR] text`
- `2024-01-15T10:30:45.123Z INFO text`
- `[ERROR] text`
- `ERROR: text`

无法匹配的行当作纯文本展示。
