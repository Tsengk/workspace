## 笔记

**2026-06-19**

JSON 解析练习。GSDS 的 device.json 是典型的嵌套 JSON 结构：`cpu_chip_vec` 是个数组，每个元素又包含 `cpu_cores` 数组。解析时关键是先理清层级关系，然后用 `dict.get(key, default)` 安全取值，避免 KeyError。

扁平化（nested → flat table）是数据处理的常见需求：把 `[{chip_name, [cores]}]` 展开为 `[{芯片, 核心}]` 的二维表，方便导出 CSV 或写入数据库。

---

**2026-06-19**

CSV 处理练习。ZMQ_TOOL 的 fault.csv 有 540 条故障码，每行有多个行为模式列（循迹/突击、自主干预等）。`csv.DictReader` 用表头做 key，遍历时每行就是一个 dict，比用索引读安全得多。

多列筛选的技巧：用列表推导 `sum(1 for f in faults if f.get(bh, '0') == '2')` 一次遍历完成多维统计。

---

**2026-06-20**

YAML 批量解析练习。用 `pathlib.Path.glob("*.yaml")` 收集所有标定文件，然后 `yaml.safe_load()` 解析每个文件。`yaml.safe_load` 比 `yaml.load` 安全，不会执行任意 Python 代码。

有意思的是同一个标定目录下不同相机的畸变参数差异很大：k2 的绝对值可差几个数量级，反映不同镜头的光学特性。

---

**2026-06-20**

格式互转综合练习。核心发现：**任何格式互转都可以抽象为两段式——源格式 → dict/list（中间表示 IR）→ 目标格式。** 这样每种格式只需要写一个 parser 和一个 serializer，N 种格式互转只需要 O(N) 个转换器而不是 O(N²)。

实践中 JSON 是最通用的中间格式：人类可读、几乎所有语言都有库支持、可以直接在代码中 `json.loads()` 得到 IR。CSV 适合给非技术人员在 Excel 中查看，YAML 适合人工编辑的配置文件。
