> 目标：说清研发流程基本节点和要求

---

## 流程节点理解

| 节点 | 输入 | 活动 | 输出 | 关键要求 |
|------|------|------|------|----------|
| 任务确认 | 任务来源（需求/缺陷/改进） | 理解目标与边界，确认范围与截止时间 | 确认结论、责任人、时间节点 | 不可模糊开工；不清楚就追问 |
| 方案评审 | 任务定义与现有代码/设计 | 出方案、评审、改方案直到通过 | 评审记录 + 最终方案 | 小任务至少口头确认，大任务必须留文档 |
| 开发 | 方案 + 编码环境 | 编码实现，保持提交原子化 | 可运行代码 + 提交记录 | 不在主干直接开发；提交颗粒度合理 |
| 自测 | 开发完成的代码 | 按自测要点执行，录屏或记录结果 | 自测记录 | 不能跳过；自测未通过不得提测/合入 |
| 发布/留痕 | 自测通过的代码 + 说明 | 合入或发布 + 更新文档和记录 | 合入记录/发布记录 + 文档更新 | 任何变更必须可追溯 |

## 各节点在本阶段培训中的对应

| 节点 | 当前培训中的体现 |
|------|-----------------|
| 任务确认 | 每周周初明确本周学习和任务目标，写周目标记录 |
| 方案评审 | 练习代码先看 workspace 参考、确认理解后再手敲；流程/规范类先阅读再确认 |
| 开发 | 在 `复盘/第2周/` 目录下手敲实现函数，每完成一题在模板中勾状态 |
| 自测 | `python3` / `node` 直接运行验证结果，对照 workspace 参考输出 |
| 留痕 | 笔记写在 `笔记/` 和 `记录/` 目录；代码提交到 git；输出物更新到模板 |

## 流程执行检查

- [ ] 每个节点的输入是否明确
- [ ] 是否有跳过的节点（尤其是方案评审和自测）
- [ ] 提交记录/文档是否与节点对应
- [ ] 是否存在"口头确认但无记录"的节点
