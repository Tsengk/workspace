# 应用工程师培训任务清单V1\.0

> 配套文件：

1. 应用工程师培训计划与验收准则V1\.0\.md

## 1\. 使用说明

- 本清单用于把培训计划拆成可执行、可跟踪、可验收的具体任务。

- 原则上按 `12 周` 执行。

- 每条任务都可直接同步到飞书文档，形成待办项。

- 每条任务完成后，应补对应输出物或链接。

- 每周或每个任务原则上都应结合现有工具软件开展，不单独做脱离业务背景的纯练习。

- 优先结合的工具软件包括：`GSDS`、`ZMQ_TOOL`、`ICAOS_TOOL`、`测试任务软件`、`MRCT`。

## 2\. 第1阶段任务清单（第1\-4周）

### 第1周

* [ ] 结合 GSDS 或 ICAOS\_TOOL 相关脚本场景，学习 python3 基础语法：变量、分支、循环、函数、列表、字典。 完成标准：能基于现有工具软件场景写出基础练习代码并运行通过。 输出物：学习笔记 1。

* [ ] 结合 MRCT、测试任务软件 或 ICAOS\_TOOL 前端场景，学习 javascript 基础语法：变量、函数、对象、数组、模块导入导出。 完成标准：能基于现有工具软件页面或逻辑场景写出基础练习代码并运行通过。 输出物：学习笔记 2。

* [ ] 完成 python3、Node\.js、Vue3、Electron 基本环境准备，并确认至少一个现有工具软件工程可运行或可阅读。 完成标准：本机可运行 Python、npm、Vue 项目、Electron Demo，并能打开至少一个现有工具软件工程。 输出物：环境检查记录。

### 第2周

* [ ] 结合 GSDS、ICAOS\_TOOL 或 测试任务软件 的真实文件、日志或配置，完成 python3 文件处理练习。 完成标准：可独立处理至少 1 组真实输入文件并输出结果。 输出物：python3 练习脚本 1。

* [ ] 结合 MRCT、ZMQ\_TOOL 或 测试任务软件 的实际代码场景，完成 javascript 异步与模块化练习。 完成标准：能独立运行并解释异步逻辑，且能对应到现有工具软件场景。 输出物：JS 练习代码。

* [ ] 学习研发流程：任务确认、评审、开发、自测、留痕流程。 完成标准：能说清研发流程基本节点和要求。 输出物：流程学习确认记录。

* [ ] 学习工作规范：闭环、留痕、沟通同步、问题升级等要求。 完成标准：能说清工作规范中的关键要求，并结合当前培训任务说明如何执行。 输出物：工作规范学习确认记录。

### 第3周

* [ ] 结合 MRCT、ICAOS\_TOOL 或 测试任务软件 的页面场景，完成 vue3 基础页面练习。 完成标准：可运行 1 个完整页面 Demo，且页面主题与现有工具软件场景相关。 输出物：Vue3 页面 Demo。

* [ ] 结合 GSDS、ICAOS\_TOOL 或 测试任务软件 的数据文件，完成 python3 数据格式处理练习。 完成标准：至少完成 1 组真实场景 json/csv/yaml 格式转换。 输出物：python3 练习脚本 2。

* [ ] 学习大模型辅助编程方法。 完成标准：能说明可用场景、风险边界和不能直接信任的内容。 输出物：大模型使用复盘 1。

* [ ] 结合本阶段任务，说明研发流程和工作规范在真实执行中的对应关系。 完成标准：至少列出任务确认、开发、自测、留痕、同步对应的实际动作。 输出物：流程与规范对应说明。

### 第4周

* [x] 结合 ICAOS_TOOL 桌面工具场景，完成 electron 日志查看器 Demo。 完成标准：能启动、展示界面并完成一个与现有工具软件场景相关的最小交互。 输出物：Electron Demo（workspaces/第4周_electron_demo/）。

* [x] 完成第一阶段学习复盘。 完成标准：形成结构化复盘，说明已掌握内容、问题和改进点。 输出物：阶段复盘 1（复盘/阶段复盘1.md）。

* [x] 汇总第一阶段验收材料。 完成标准：学习笔记、练习脚本、Demo 和学习确认记录齐全。 输出物：阶段验收材料 1（验收/阶段验收材料1.md）。

## 3\. 第2阶段任务清单（第5\-8周）

### 第5周

* [ ] 从 GSDS、ZMQ\_TOOL、ICAOS\_TOOL、测试任务软件、MRCT 中选择一个真实工具类小功能作为实战任务。 完成标准：明确任务来源、目标、边界、截止时间，并明确对应工具软件。 输出物：任务定义记录。

* [ ] 对真实小功能进行最小任务拆解，并标明涉及的工具软件模块或页面。 完成标准：明确实现步骤、验证点、交付结果和影响范围。 输出物：功能拆解记录。

### 第6周

* [ ] 完成真实小功能第一版开发。 完成标准：功能可运行，核心路径可执行。 输出物：代码提交记录。

* [ ] 为该功能编写自测记录。 完成标准：自测记录包含输入、操作步骤、结果和问题。 输出物：自测记录 1。

* [ ] 结合真实小功能，检查工作规范和研发流程执行情况。 完成标准：能说明该功能的任务来源、验证方式、留痕位置和同步对象。 输出物：流程执行检查记录 1。

### 第7周

* [ ] 根据自测或反馈对小功能进行联调或补充优化。 完成标准：功能行为与目标一致。 输出物：二次提交记录。

* [ ] 编写最小使用说明。 完成标准：其他人可按说明完成运行和使用。 输出物：使用说明 1。

### 第8周

* [ ] 完成第二阶段复盘。 完成标准：总结真实小功能开发中的问题、经验和不足。 输出物：阶段复盘 2。

* [ ] 汇总第二阶段验收材料。 完成标准：功能实现、自测、说明和结果留痕齐全。 输出物：阶段验收材料 2。

## 4\. 第3阶段任务清单（第9\-12周）

### 第9周

* [ ] 从 GSDS、ZMQ\_TOOL、ICAOS\_TOOL、测试任务软件、MRCT 中选择一个真实工具类问题作为独立闭环对象。 完成标准：明确问题现象、范围、责任边界和对应工具软件。 输出物：问题定义记录。

* [ ] 制定问题分析计划，并写清对应工具软件中的定位入口。 完成标准：明确分析路径、验证方式和相关模块/页面/脚本。 输出物：分析计划记录。

### 第10周

* [ ] 完成真实问题根因分析。 完成标准：形成可自证的根因判断。 输出物：根因分析记录。

* [ ] 完成问题修复或规避实现。 完成标准：修复代码可运行。 输出物：修复提交记录。

### 第11周

* [ ] 对修复结果进行验证。 完成标准：明确验证步骤、结果和残留风险。 输出物：验证记录。

* [ ] 补齐问题闭环留痕。 完成标准：形成包含现象、根因、修改、验证、结论的完整记录。 输出物：问题闭环记录。

* [ ] 结合真实问题闭环，检查工作规范和研发流程执行情况。 完成标准：能说明问题升级、留痕、验证、同步是否符合要求。 输出物：流程执行检查记录 2。

### 第12周

* [ ] 完成最终复盘与分享。 完成标准：总结 12 周培训收获并完成一次组内分享。 输出物：最终复盘 \+ 分享材料。

* [ ] 汇总最终验收材料。 完成标准：全部阶段材料齐全，可对照培训计划逐项检查。 输出物：最终验收材料。

## 5\. 周度检查任务

* [ ] 每周周初明确本周学习和任务目标。 完成标准：本周任务清单已明确。 输出物：周目标记录。

* [ ] 每周周中完成一次辅导或检查。 完成标准：有主管或辅导人检查记录。 输出物：检查记录。

* [ ] 每周周末输出本周结果。 完成标准：至少有笔记、代码、Demo、记录中的一种产出。 输出物：周结果记录。

## 6\. 阶段验收任务

* [ ] 第4周完成第一阶段验收。 完成标准：基础学习和练习产出齐套。 输出物：第一阶段验收结论。

* [ ] 第8周完成第二阶段验收。 完成标准：真实小功能闭环齐套。 输出物：第二阶段验收结论。

* [ ] 第12周完成最终验收。 完成标准：真实问题闭环、复盘和分享材料齐套。 输出物：最终验收结论。

* [ ] 各阶段验收时同步检查工作规范、研发流程执行情况。 完成标准：不能只看代码和功能结果，还要检查任务闭环、验证、留痕和同步是否达标。 输出物：规范与流程验收记录。

## 7\. 达标判断

* [ ] 任务完成率达到 85% 以上。

* [ ] 完成不少于 4 份学习/复盘材料。

* [ ] 完成不少于 2 个练习类或演示类产出。

* [ ] 完成不少于 1 个真实功能闭环。

* [ ] 完成不少于 1 个真实问题闭环。

* [ ] 能提供完整证据链，而不是只口头说明。
