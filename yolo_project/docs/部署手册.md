# 玫瑰病害分类模型 — 部署手册

## 项目概述

基于 YOLOv8n-cls 的玫瑰叶片病害图像分类模型，支持 8 种病害类别识别。

## 环境要求

| 项 | 最低配置 | 推荐配置 |
|---|---------|---------|
| Python | 3.8+ | 3.10+ |
| 内存 | 8 GB | 16 GB |
| 磁盘 | 5 GB | 10 GB |
| GPU | 无（CPU 可跑） | NVIDIA 4GB+ 显存 |

## 目录结构

```
yolo_project/
├── train.py              # 训练脚本
├── predict.py            # 预测脚本
├── datasets/             # 数据集
│   ├── train/            #   训练集（8 个子目录 = 8 个类别）
│   │   ├── black_spot/
│   │   ├── downy_mildew/
│   │   ├── fresh_leaf/
│   │   ├── insect_hole/
│   │   ├── insects_infected/
│   │   ├── powdery_mildew/
│   │   ├── rose_mosaic/
│   │   └── rose_rust/
│   └── val/              #   验证集（同上结构）
├── runs/                 # 训练输出
│   ├── rose_quick/       #   快速验证
│   ├── rose_disease/     #   正式训练
│   └── predict/          #   预测结果图
│   └── rose_disease/
│       └── weights/
│           └── best.pt   # 最佳模型
└── docs/                 # 文档
```

## 部署步骤

### 1. 克隆项目

```bash
git clone <仓库地址>
cd yolo_project
```

### 2. 准备数据集

确保 `datasets/` 目录结构正确：

```
datasets/
├── train/
│   ├── 类别A/   (图片直接放里面)
│   └── 类别B/
└── val/
    ├── 类别A/
    └── 类别B/
```

新增类别只需在 `train/` 和 `val/` 下各建一个文件夹，放入图片即可。

### 3. 创建虚拟环境

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 4. 安装依赖

```bash
pip install ultralytics matplotlib
```

如果出现 Polars CPU 警告，安装兼容版本：

```bash
pip uninstall polars -y
pip install polars-lts-cpu
```

或者跳过检查：

```bash
export POLARS_SKIP_CPU_CHECK=1
```

### 5. 训练

**快速验证**（3 epochs, 224px, ~10 分钟）：

```bash
python train.py --quick
```

**正式训练**（100 epochs, 640px, ~8-12 小时）：

```bash
python train.py
```

两种模式对比：

| 模式 | epochs | imgsz | 耗时 | 用途 |
|------|--------|-------|------|------|
| `--quick` | 3 | 224 | ~10 分钟 | 验证流程无报错 |
| 默认 | 100 | 640 | ~8-12 小时 | 产出可用模型 |

关键参数（可在 `train.py` 中修改）：

| 参数 | 含义 | 建议值 |
|-----|------|-------|
| model | 预训练权重 | yolov8n-cls.pt（快）/ yolov8s-cls.pt（准）|
| data | 数据集路径 | datasets/ |
| epochs | 训练轮数 | 快速: 3, 正式: 50-100 |
| imgsz | 图片尺寸 | 快速: 224, 正式: 640 |
| batch | 批次大小 | CPU: 8, GPU: 32 |
| device | 设备 | cpu / 0（GPU编号）|

快速验证模型保存在 `runs/rose_quick/weights/best.pt`。
正式训练模型保存在 `runs/rose_disease/weights/best.pt`。

### 6. 验证准确率

```bash
# 快速模型
python predict.py --val --model runs/rose_quick/weights/best.pt

# 正式模型
python predict.py --val

# YOLO 内置验证
yolo classify val model=runs/rose_disease/weights/best.pt data=datasets/
```

### 7. 预测

```bash
# 单张图片（终端输出结果）
python predict.py /path/to/leaf.jpg

# 单张图片（结果画在图上并保存）
python predict.py /path/to/leaf.jpg --save

# 整个文件夹
python predict.py /path/to/folder/

# 批量预测并保存结果图
python predict.py datasets/val/black_spot/ --save

# 指定模型
python predict.py leaf.jpg --model runs/rose_quick/weights/best.pt

# YOLO CLI 方式
yolo classify predict model=runs/rose_disease/weights/best.pt source=图片路径
```

`--save` 结果图保存到 `runs/predict/`：
- 单张图：`runs/predict/文件名_pred.jpg`
- 批量预测：`runs/predict/真实类别/文件名_pred.jpg`

### 8. 常见问题

**Q: Polars CPU 警告**
```bash
export POLARS_SKIP_CPU_CHECK=1
```
或 `pip uninstall polars -y && pip install polars-lts-cpu`

**Q: Shared Memory 报错**
修改 `train.py` 中 `workers=0`

**Q: 显存不足**
减小 `batch=4` 或 `batch=2`

**Q: 训练太慢**
先用 `--quick` 快速验证，确认无误后再正式训练

**Q: 新增类别**
在 `datasets/train/` 和 `datasets/val/` 下各建新文件夹放入图片，重新训练即可
