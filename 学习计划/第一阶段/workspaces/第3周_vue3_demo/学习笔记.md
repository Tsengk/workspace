## 笔记

**2026-06-19**

Vue3 项目初始化。用 Vite 做构建工具，比 Vue CLI 快很多——原生 ESM 开发服务器、毫秒级热更新。`vite.config.js` 只需要引入 `@vitejs/plugin-vue` 插件即可支持 `.vue` 单文件组件。

Vue3 的 `<script setup>` 是推荐的 Composition API 写法：更少的样板代码，直接写顶层变量就自动暴露给模板，`defineProps` / `defineEmits` 编译时宏不需要导入。

---

**2026-06-19**

组件设计思路。把页面拆成四个组件：
- `App.vue` — 布局壳（顶栏 + 左右分栏）
- `TaskList.vue` — 任务卡片列表 + 筛选栏（纯展示 + 筛选逻辑）
- `TaskDetail.vue` — 右侧详情面板（条件渲染 `v-if="task"`）
- `TaskForm.vue` — 新建任务表单（用 `v-if` 控制展开/收起）

组件通信只用了 props 向下 + emit 向上，没有引入 Vuex/Pinia——对于这个规模的 Demo，一个 `reactive` 单例 store 就够用，也是理解响应式原理的好练习。

---

**2026-06-20**

响应式数据设计。用 `reactive()` 创建一个 store 对象，把筛选逻辑写成 `get filteredTasks()` ——这是 Vue3 computed getter 的替代写法，在 reactive 对象里定义 getter 同样具备缓存和自动追踪依赖的能力。

`v-model` 双向绑定的本质是 `:value` + `@input` 的语法糖。在表单组件里，`v-model="form.name"` 让输入框和 reactive 对象自动同步，比手动 `e.target.value` 干净很多。

---

**2026-06-20**

Scoped CSS 踩坑和收获。Vue SFC 的 `<style scoped>` 自动给每个元素加 `data-v-xxx` 属性选择器，组件样式完全隔离。但注意：scoped 不影响子组件的根节点——这是有意设计的，让父组件可以给子组件根元素加布局样式。

另外 CSS 的 `.status-pending` 等动态 class 结合 `:class` 绑定，替代了传统的 className 字符串拼接，配合 computed 可以很优雅地做条件样式。
